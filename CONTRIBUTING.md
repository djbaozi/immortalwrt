# 贡献指南

感谢您对 ImmortalWrt 的兴趣！我们欢迎各种形式的贡献。

## 🚀 如何贡献

### 1. 报告问题
如果您发现了bug或有功能建议：
- 使用 [Issues 页面](https://github.com/djbaozi/immortalwrt/issues) 提交问题
- 选择正确的模板（Bug报告/功能请求/问题咨询）
- 提供尽可能详细的信息，包括：
  - ImmortalWrt版本
  - 设备型号
  - 复现步骤
  - 错误日志

### 2. 提交代码更改
1. **Fork 仓库** - 点击右上角的 Fork 按钮
2. **创建分支** - `git checkout -b feature/your-feature-name`
3. **提交更改** - `git commit -m "Add some feature"`
4. **推送到分支** - `git push origin feature/your-feature-name`
5. **创建 Pull Request** - 在 GitHub 上创建 PR

### 3. 编写文档
- 改进现有文档
- 添加使用教程
- 翻译文档
- 修复拼写错误

### 4. 回答问题
- 帮助其他用户解决问题
- 参与 Issues 讨论
- 分享使用经验

## 📝 代码规范

### 提交信息规范
使用清晰、简洁的提交信息：
```
类型(范围): 简短描述

详细描述（可选）

关联的 Issue: #123
```

**类型**：
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建过程或辅助工具变动

### 代码风格
- 遵循 OpenWrt 的代码风格
- 使用有意义的变量名
- 添加必要的注释
- 确保代码通过 CI 检查

## 🧪 测试要求

### 提交前检查
1. 代码编译通过
2. 基本功能测试正常
3. 不破坏现有功能
4. 通过 GitHub Actions CI 检查

### 测试环境
- 建议在实际设备上测试
- 提供测试结果和日志
- 说明测试环境和步骤

## 🤝 社区准则

### 沟通准则
- 保持友好和尊重
- 使用清晰的语言
- 提供建设性反馈
- 帮助新贡献者

### 行为准则
请遵守我们的 [行为准则](CODE_OF_CONDUCT.md)。

## 🎯 贡献优先级

### 高优先级
1. 安全漏洞修复
2. 严重bug修复
3. 文档改进

### 中优先级
1. 新功能开发
2. 性能优化
3. 测试覆盖

### 低优先级
1. 代码重构
2. 界面美化
3. 小功能改进

## 📞 获取帮助

- **Issues**: 提交问题或提问
- **Discussions**: 参与技术讨论
- **Wiki**: 查看使用文档
