# 安全策略

## 报告安全漏洞

我们非常重视 ImmortalWrt 的安全性。如果您发现了安全漏洞，请立即报告。

### 报告方式

**请勿通过公开的 Issues 报告安全漏洞！**

请通过以下方式私下报告：

1. **发送邮件至**：项目维护者
2. **使用安全联系人**：如果配置了 GitHub 安全通告，请使用该功能

### 报告内容

请在报告中包含以下信息：

- **漏洞描述**：详细说明漏洞的性质
- **影响范围**：哪些版本/设备受影响
- **复现步骤**：如何复现该漏洞
- **潜在影响**：漏洞可能造成的危害
- **建议修复**：如果有修复建议

### 处理流程

1. **确认收到**：我们将在 48 小时内确认收到报告
2. **调查评估**：评估漏洞的严重性和影响范围
3. **制定方案**：制定修复方案和时间表
4. **发布修复**：发布安全更新
5. **公开披露**：在修复发布后公开披露（根据严重程度）

## 安全更新策略

### 严重性等级

| 等级 | 描述 | 响应时间 |
|------|------|----------|
| **严重** | 可远程执行代码、获取 root 权限等 | 24小时内响应，7天内修复 |
| **高危** | 信息泄露、拒绝服务等 | 48小时内响应，14天内修复 |
| **中危** | 权限提升、配置错误等 | 7天内响应，30天内修复 |
| **低危** | 轻微信息泄露、配置问题等 | 14天内响应，酌情修复 |

### 更新发布

安全更新将通过以下方式发布：

1. **GitHub Releases**：发布新版本
2. **安全通告**：通过 GitHub 安全通告功能
3. **项目公告**：在 README 或 Wiki 中公告

## 安全最佳实践

### 对于用户

1. **及时更新**：定期更新到最新版本
2. **最小权限**：使用最小必要权限运行
3. **网络隔离**：将路由器放在受保护的网络环境中
4. **强密码**：使用强密码保护管理界面
5. **禁用不需要的服务**：关闭不需要的网络服务

### 对于开发者

1. **代码审查**：所有代码变更都需要审查
2. **依赖更新**：定期更新依赖库
3. **安全测试**：进行安全测试和代码审计
4. **最小权限原则**：代码以最小必要权限运行

## 依赖安全

### 上游依赖
本项目基于 OpenWrt，继承其安全更新：
- 定期同步上游安全修复
- 关注上游安全通告

### 第三方包
- 定期检查第三方包的安全更新
- 及时更新有安全问题的包
- 移除不再维护的包
